# 游戏逻辑测试客户端

## 功能介绍

游戏逻辑测试客户端用于测试游戏服务器的功能，包括：

1. **HTTP登录验证**：通过验证服务器进行登录，获取session和token
2. **获取角色列表**：获取已创建的游戏角色列表
3. **角色选择**：如果只有一个角色，自动选择；如果有多个角色，让用户选择
4. **角色创建**：如果没有角色，创建新的游戏角色
5. **游戏功能测试**：等待更多的游戏功能测试

## 执行程序位置

测试执行程序位于 `test/bin` 目录下：
- Windows: `test/bin/gamelogic_test.exe`

## 运行方式

### 基本运行
直接运行执行程序，使用默认参数：

```bash
test/bin/gamelogic_test.exe
```

### 使用命令行参数
可以通过命令行参数指定验证服务器URL、游戏服务器URL、用户名、密码和角色名称：

```bash
test/bin/gamelogic_test.exe -auth=http://localhost:8080 -game=http://localhost:8081 -username=za_admin -password=za_admin
```

## 可用的命令行参数

| 参数 | 描述 | 默认值 |
|------|------|--------|
| `-auth` | 验证服务器URL | `http://localhost:8080` |
| `-game` | 游戏服务器URL | `http://localhost:8081` |
| `-username` | 用户名 | `za_admin` |
| `-password` | 密码 | `za_admin` |
| `-actor` | 角色名称（可选） | 空 |
| `help` | 打印帮助信息 | - |

## 测试流程

执行程序会按照以下流程进行测试：

1. **步骤1：登录验证**
   - 向验证服务器发送登录请求
   - 获取session和token

2. **步骤2：获取角色列表**
   - 向游戏服务器发送获取角色列表请求
   - 显示所有已创建的角色

3. **步骤3：角色选择或创建**
   - 如果没有角色，创建新角色
   - 如果只有一个角色，自动选择
   - 如果有多个角色：
     - 如果参数提供了角色名称，尝试查找并选择
     - 如果没有提供角色名称，让用户选择

4. **步骤4：等待更多游戏功能测试**
   - 显示当前角色信息
   - 等待后续的游戏功能测试

## 测试输出

执行程序会在控制台打印详细的测试信息，包括：

- 命令行参数值
- 每个测试步骤的执行情况
- 登录结果（session和token）
- 角色列表
- 角色选择或创建结果
- 当前角色信息

## 注意事项

- 确保验证服务器（ssoauth）正在运行，并且监听在指定的URL上
- 确保游戏服务器正在运行，并且监听在指定的URL上
- 登录需要使用正确的用户名和密码
- 如果没有角色，会提示创建新角色
- 如果有多个角色，需要用户选择或通过参数指定角色名称
- 测试客户端目前只实现了基本的登录和角色管理功能，后续会添加更多的游戏功能测试
